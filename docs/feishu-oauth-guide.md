# 飞书 OAuth 登录说明

## 问题背景

企业自建应用无法直接访问用户个人创建的多维表，即使有正确的 App Token 和所有权限。

## 解决方案

通过飞书 OAuth 授权登录，系统可以使用你的身份访问你的多维表。

## 使用步骤

### 1. 使用飞书登录

1. 打开飞书集成配置对话框
2. 点击底部的 **"使用飞书登录"** 按钮
3. 跳转到飞书授权页面
4. 使用飞书账号登录并授权
5. 授权成功后，会自动跳转回系统

### 2. 测试访问个人多维表

1. 登录成功后，在飞书集成配置对话框的 **"同步设置"** 标签页
2. 点击 **"测试个人表"** 按钮
3. 输入你个人创建的多维表的 App Token 和 Table ID
4. 点击 **"开始测试"**
5. 如果能成功读取到数据，说明 OAuth 登录成功！

### 3. 配置飞书集成

1. 在 **"应用配置"** 标签页，填写 App ID 和 App Secret（可选）
2. 在 **"表格配置"** 标签页，填写表格 ID
3. 点击 **"保存配置"**

## 优势

- ✅ 可以访问你个人创建的所有多维表
- ✅ 使用你的权限和设置
- ✅ 无需额外的授权配置
- ✅ 无需创建企业应用

## 注意事项

1. **OAuth 令牌有效期**：用户令牌有有效期，过期后需要重新登录
2. **权限范围**：系统会请求读取多维表的权限
3. **安全性**：令牌存储在浏览器 localStorage 中，请勿在公共设备上使用

## 常见问题

### Q: 为什么需要登录飞书？

A: 企业自建应用只能访问通过它自己创建的多维表，无法直接访问个人多维表。通过 OAuth 登录，系统可以使用你的身份访问你的多维表。

### Q: 登录后能看到哪些数据？

A: 系统只能看到你有权限访问的多维表和记录。

### Q: 登录后我的数据安全吗？

A: 系统只请求读取多维表的权限，不会修改或删除你的数据。

### Q: 如何退出登录？

A: 在飞书集成配置对话框中，会显示当前登录状态。如果需要退出，可以清除浏览器的 localStorage，或者联系管理员。

## 技术细节

### OAuth 流程

1. 用户点击登录按钮
2. 系统跳转到飞书授权页面
3. 用户授权
4. 飞书重定向回系统，带上授权码
5. 系统用授权码换取 access_token
6. 使用 access_token 访问用户的多维表

### 权限范围

系统请求的权限：
- `bitable:app` - 多维表操作
- `bitable:app:readonly` - 多维表只读访问

### 令牌管理

- 用户令牌存储在浏览器 localStorage
- 应用令牌存储在服务端（如果使用企业应用）
- 令牌有有效期，过期后需要重新获取
