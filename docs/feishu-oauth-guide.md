# 飞书扫码登录说明

## 问题背景

企业自建应用无法直接访问用户个人创建的多维表，即使有正确的 App Token 和所有权限。

## 解决方案

通过飞书**扫码登录**授权，系统可以使用你的身份访问你的多维表。

## 使用步骤

### 1. 扫码登录

1. 打开飞书集成配置对话框
2. 点击底部的 **"扫码登录"** 按钮
3. 点击 **"生成扫码登录二维码"** 按钮
4. 使用飞书 App 扫描二维码
5. 在飞书 App 中确认授权
6. 授权成功后，自动登录系统

### 2. 测试访问个人多维表

1. 登录成功后，在飞书集成配置对话框的 **"同步设置"** 标签页
2. 点击 **"测试个人表"** 按钮
3. 输入你个人创建的多维表的 App Token 和 Table ID
4. 点击 **"开始测试"**
5. 如果能成功读取到数据，说明登录成功！

### 3. 配置飞书集成

1. 在 **"应用配置"** 标签页，填写 App ID 和 App Secret（可选）
2. 在 **"表格配置"** 标签页，填写表格 ID
3. 点击 **"保存配置"**

## 优势

- ✅ 可以访问你个人创建的所有多维表
- ✅ 使用你的权限和设置
- ✅ 无需额外的授权配置
- ✅ 无需创建企业应用
- ✅ 安全快捷，扫码即可登录

## 注意事项

1. **令牌有效期**：用户令牌有有效期，过期后需要重新登录
2. **权限范围**：系统会请求读取多维表的权限
3. **安全性**：令牌存储在浏览器 localStorage 中，请勿在公共设备上使用
4. **二维码有效期**：生成的二维码有有效期（通常 5 分钟），过期后需要重新生成

## 常见问题

### Q: 为什么需要扫码登录？

A: 企业自建应用只能访问通过它自己创建的多维表，无法直接访问个人多维表。通过扫码登录，系统可以使用你的身份访问你的多维表。

### Q: 登录后能看到哪些数据？

A: 系统只能看到你有权限访问的多维表和记录。

### Q: 登录后我的数据安全吗？

A: 系统只请求读取多维表的权限，不会修改或删除你的数据。

### Q: 如何退出登录？

A: 在飞书集成配置对话框或扫码登录页面，点击 **"退出登录"** 按钮即可。

### Q: 二维码过期了怎么办？

A: 二维码通常有效期为 5 分钟，过期后点击 **"生成扫码登录二维码"** 按钮重新生成即可。

### Q: 扫码后没有反应？

A: 请确保：
1. 已在飞书 App 中打开扫码功能
2. 网络连接正常
3. 二维码没有过期

## 技术细节

### 扫码登录流程

1. 系统生成二维码
2. 用户使用飞书 App 扫码
3. 飞书 App 显示授权页面
4. 用户确认授权
5. 系统获取用户访问令牌（user_access_token）
6. 使用访问令牌访问用户的多维表

### 权限范围

系统请求的权限：
- 读取多维表数据

### 令牌管理

- 用户令牌存储在浏览器 localStorage
- 令牌有有效期，过期后需要重新获取
- 退出登录时会清除令牌

### API 接口

- **生成二维码**：`POST /api/feishu/oauth/qr-code`
- **查询扫码状态**：`POST /api/feishu/oauth/qr-status`
- **读取表格（用户令牌）**：`POST /api/feishu/user/read-table`

## 相关页面

- 飞书扫码登录页面：`/feishu-oauth`
- 用户令牌测试页面：`/feishu-user-token-test`
- 飞书集成配置：打开飞书集成对话框

## 错误处理

### 错误码 20043

**错误信息**：`bitable:app,bitable:app:readonly 有误`

**原因**：权限作用域格式错误

**解决**：使用扫码登录而不是授权码登录，扫码登录不需要指定权限作用域。

### 二维码过期

**错误信息**：二维码已过期

**解决**：重新生成二维码

### 扫码无响应

**解决**：
1. 检查网络连接
2. 确认二维码没有过期
3. 重新生成二维码

## 对比：扫码登录 vs 授权码登录

| 特性 | 扫码登录 | 授权码登录 |
|------|---------|-----------|
| 使用场景 | PC 端应用 | 网页应用 |
| 安全性 | 高（扫码确认） | 中（输入密码） |
| 用户体验 | 简单快捷 | 需要输入密码 |
| 权限作用域 | 不需要指定 | 需要正确指定 |
| 推荐度 | ⭐⭐⭐⭐⭐ | ⭐⭐⭐ |

**推荐使用扫码登录**，更安全、更简单！
