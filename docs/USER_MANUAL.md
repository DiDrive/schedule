# 唯变项目排期系统 - 用户手册

## 一、系统简介

唯变项目排期系统是一个智能化的项目排期管理工具，帮助团队高效管理项目任务、合理分配人员资源、自动生成排期计划。

### 核心价值

- **智能排期**：系统自动计算任务时间，无需手动安排
- **资源优化**：自动分配最合适的人员，避免资源冲突
- **飞书集成**：数据一键加载、一键同步，告别重复录入
- **可视化管理**：甘特图直观展示项目进度和人员安排

## 三、快速开始

### 第一步：配置飞书集成

在使用系统前，需要先配置飞书多维表的连接信息。

1. 点击右上角「飞书配置」按钮
2. 填写以下信息：
   - **App ID**：飞书应用的唯一标识
   - **App Secret**：飞书应用的密钥
   - **多维表 Token**：多维表的访问令牌
3. 选择数据源模式：
   - **需求表模式**：从需求表加载任务数据
   - **传统模式**：从项目表和任务表加载数据
4. 填写各数据表的 ID
5. 点击「测试连接」验证配置是否正确
6. 保存配置

> 💡 提示：如果不确定如何获取这些信息，请点击「配置助手」查看详细指南。

### 第二步：加载数据

配置完成后，从飞书加载数据：

1. 点击「从飞书加载」按钮
2. 系统自动读取人员、项目、任务数据
3. 加载完成后页面自动刷新

### 第三步：管理任务

在任务列表中可以：

- **查看任务**：查看从飞书加载的所有任务
- **编辑任务**：修改任务名称、工时、类型、优先级等
- **设置依赖**：指定任务的前置依赖关系
- **指定负责人**：手动指定任务负责人，或让系统自动分配

### 第四步：生成排期

1. 点击「生成排期」按钮
2. 系统自动：
   - 为未分配的任务分配合适的人员
   - 计算每个任务的开始和结束时间
   - 检测资源冲突
3. 查看排期结果：
   - **甘特图**：可视化展示任务时间线
   - **冲突列表**：查看资源冲突情况

### 第五步：同步到飞书

确认排期结果无误后，同步到飞书多维表：

1. 点击「同步到飞书」按钮
2. 等待同步完成
3. 查看同步结果统计

## 四、功能详解

### 4.1 资源管理

资源即团队成员，每个资源包含以下信息：

| 字段 | 说明 |
|------|------|
| 姓名 | 人员姓名 |
| 工作类型 | 平面 / 后期 / 物料 |
| 效率系数 | 工作效率（影响任务完成时间） |
| 可用性 | 人员的可用程度 |

**工作类型说明**：
- **平面**：负责平面设计类任务
- **后期**：负责后期制作类任务
- **物料**：负责物料准备相关任务

### 4.2 任务管理

#### 任务基本信息

每个任务包含以下核心属性：

| 字段 | 说明 | 可选值 |
|------|------|--------|
| 任务名称 | 任务的标题 | 自由输入 |
| 任务类型 | 任务的工作类型 | 平面 / 后期 / 物料 / 复合 |
| 预估工时 | 完成任务需要的时间 | 数字（小时） |
| 优先级 | 任务的重要程度 | 紧急 / 高 / 普通 / 低 |
| 截止日期 | 任务必须完成的时间 | 日期选择 |
| 所属项目 | 任务归属的项目 | 下拉选择 |
| 指定负责人 | 手动指定任务负责人 | 下拉选择 |

#### 任务类型详解

- **平面**：平面设计类任务，如概念设计、分镜设计、海报制作等
- **后期**：后期制作类任务，如视频剪辑、特效制作、调色等
- **物料**：物料准备类任务，如道具采购、服装准备等
- **复合**：同时包含平面和后期工作的任务，系统会自动拆分

#### 优先级说明

- **紧急**：最优先处理，资源分配时优先考虑
- **高**：重要任务，尽量优先完成
- **普通**：常规任务，按正常顺序安排
- **低**：不紧急的任务，可以延后处理

#### 依赖关系

依赖关系用于指定任务的执行顺序：

- 如果任务B依赖任务A，则任务B必须等任务A完成后才能开始
- 一个任务可以依赖多个前置任务
- 依赖关系形成任务链条，系统会自动计算正确的开始时间

### 4.3 复合任务

#### 什么是复合任务？

复合任务是指同时包含平面工作和后期工作的任务。例如：
- 一个视频制作任务，既需要平面设计分镜（4小时），又需要后期剪辑（8小时）

#### 复合任务的设置

创建复合任务时需要设置：

1. **平面工时**：平面工作需要的预估时间
2. **后期工时**：后期工作需要的预估时间
3. **子任务依赖模式**：
   - **并行**：平面和后期工作可以同时进行
   - **串行**：后期工作必须等平面工作完成后才能开始

#### 复合任务的处理

系统会自动将复合任务拆分为：
- 一个父任务（原始任务）
- 一个平面子任务（分配给平面人员）
- 一个后期子任务（分配给后期人员）

### 4.4 排期生成

#### 排期算法说明

系统按照以下规则自动排期：

**工作时间安排**：
- 工作日：周一至周五
- 工作时间：上午 9:30 - 12:00，下午 13:30 - 18:30
- 午休时间：12:00 - 13:30（不计入工作时间）
- 周末：自动跳过

**资源分配原则**：
1. 被其他任务依赖的任务优先分配资源（避免阻塞后续任务）
2. 优先级高的任务优先安排
3. 任务类型匹配人员工作类型（平面任务分配给平面人员）
4. 负载均衡，避免所有人都挤在一个项目

**时间计算规则**：
- 根据依赖关系确定最早开始时间
- 根据工时和人员效率计算结束时间
- 自动跳过非工作时间

#### 排期结果查看

排期完成后，可以通过以下方式查看结果：

**甘特图视图**：
- 横轴为时间线
- 每个任务显示为一个条形
- 不同颜色代表不同项目
- 可以看到任务之间的依赖关系

**日历视图**：
- 按日期查看任务安排
- 显示每天的任务列表

**冲突列表**：
- 显示资源时间冲突的情况
- 提供冲突处理建议

### 4.5 任务筛选

系统提供多种筛选方式，帮助你快速找到需要的任务：

#### 项目筛选

- 在任务列表上方选择「项目筛选」下拉框
- 选择特定项目，查看该项目下的所有任务
- 选择「全部项目」显示所有任务

#### 负责人筛选

- 在任务列表上方选择「负责人筛选」下拉框
- 选择特定人员，查看该人员负责的所有任务
- 系统会自动匹配：
  - 已分配给该人员的任务（`assignedResources`）
  - 指定该人员负责的任务（`fixedResourceId`）
  - 复合任务中指定该人员负责的平面或后期部分

#### 任务类型筛选

- 在任务列表上方点击类型标签页
- 可选择：全部 / 平面 / 后期 / 物料

#### 组合筛选

以上筛选条件可以组合使用：
- 例如：选择「宣传片项目」+「张设计师」，查看宣传片项目中张设计师负责的任务

> 💡 提示：筛选结果会同步应用到甘特图和日历视图中。

### 4.6 飞书集成

#### 数据源模式

系统支持两种数据源模式：

**需求表模式（推荐）**：
- 从需求表1和需求表2加载任务
- 适用于以需求为导向的项目管理
- 每个需求作为一条任务加载

**传统模式**：
- 从项目表和任务表分别加载数据
- 适用于传统的项目-任务层级管理

#### 数据同步机制

**从飞书加载**：
- 自动转换字段格式
- 数据保存到本地，刷新不丢失

**同步到飞书**：
- 显示同步进度
- 返回成功/失败统计

## 五、项目模板

### 5.1 模板介绍

项目模板是预定义的任务结构，可以快速创建标准化项目。

### 5.2 使用模板创建项目

1. 点击「从模板创建」按钮
2. 选择合适的模板
3. 填写项目信息：
   - 项目名称
   - 项目优先级
   - 开始日期
   - 资源池（参与项目的人员）
4. 系统自动创建任务列表
5. 根据实际情况调整任务

### 5.3 自定义模板

可以创建和编辑自定义模板：

1. 点击「模板管理」进入模板编辑器
2. 添加、编辑、删除任务
3. 设置任务属性：
   - 任务名称和描述
   - 任务类型
   - 预估工时
   - 优先级
   - 依赖关系
4. 保存模板供后续使用

## 六、常见场景操作

### 场景一：新建项目并排期

1. 从模板创建项目（或手动添加任务）
2. 设置任务的截止日期和优先级
3. 设置任务间的依赖关系
4. 点击「生成排期」
5. 查看排期结果，处理冲突
6. 同步到飞书

### 场景二：调整已有排期

1. 从飞书加载现有数据
2. 修改任务属性（如工时、负责人）
3. 点击「生成排期」重新计算
4. 查看新的排期结果
5. 同步到飞书更新排期表

### 场景三：处理复合任务

1. 创建任务时选择「复合」类型
2. 设置平面工时和后期工时
3. 选择子任务依赖模式（并行/串行）
4. 生成排期时系统自动拆分
5. 查看拆分后的子任务安排

### 场景四：处理资源冲突

1. 生成排期后查看冲突列表
2. 分析冲突原因：
   - 同一人同一时间有多个任务
   - 任务依赖关系导致的时间重叠
3. 解决方案：
   - 手动指定其他负责人
   - 调整任务优先级
4. 重新生成排期验证

## 七、注意事项

### 数据安全

- 飞书配置信息保存在浏览器本地，不会上传到服务器
- 排期数据保存在浏览器本地，刷新页面不会丢失
- 清除浏览器缓存会导致数据丢失，请及时同步到飞书

### 排期建议

- 合理设置任务工时，避免过高或过低
- 正确设置依赖关系，避免循环依赖
- 及时更新任务状态，保持排期准确性
- 处理资源冲突，避免人员过载

### 最佳实践

1. **定期同步**：排期变更后及时同步到飞书
2. **合理分配**：不要让一个人承担过多任务
3. **预留缓冲**：给紧急任务预留一些时间缓冲
4. **及时更新**：任务完成后更新状态，便于重新排期

## 八、常见问题

### Q1：从飞书加载失败怎么办？

可能原因：
- 飞书配置信息不正确
- 应用没有多维表访问权限
- 表 ID 填写错误
- 网络连接问题

解决方法：
1. 点击「配置助手」检查配置是否正确
2. 确认飞书应用有多维表的读写权限
3. 检查各表的 ID 是否正确
4. 尝试点击「测试连接」验证

### Q2：排期结果不符合预期怎么办？

可能原因：
- 任务依赖关系设置不正确
- 任务类型与人员工作类型不匹配
- 任务工时设置不合理

解决方法：
1. 检查任务依赖关系是否符合实际流程
2. 确认任务类型（平面/后期）与人员匹配
3. 调整任务工时和优先级
4. 手动指定关键任务的负责人

### Q3：同步到飞书失败怎么办？

可能原因：
- 排期表 ID 未配置或错误
- 应用没有创建记录的权限
- 数据格式不符合要求

解决方法：
1. 确认排期表 ID 已正确配置
2. 检查飞书应用的权限设置
3. 查看错误详情，根据提示修复

### Q4：如何查看某个人负责的所有任务？

方法：
1. 在任务列表上方的筛选区域，找到「负责人筛选」下拉框
2. 选择要查看的人员
3. 任务列表会自动筛选出该人员负责的所有任务（包括已分配和指定的任务）
4. 排期结果（甘特图/日历视图）也会同步筛选

> 💡 提示：可以同时使用项目筛选和负责人筛选，查看某个项目中某个人负责的任务。

### Q5：复合任务拆分后如何修改？

方法：
1. 编辑原始任务（父任务）的属性
2. 重新生成排期，系统会自动更新拆分结果
3. 不要直接编辑拆分后的子任务，会被覆盖

### Q6：为什么任务开始时间比我预期的晚？

可能原因：
- 任务依赖其他未完成的任务
- 分配的人员已经排满其他任务
- 任务安排在周末，自动顺延到工作日

解决方法：
1. 检查任务的依赖关系
2. 查看人员的任务负载
3. 考虑调整任务优先级或更换负责人

## 九、术语表

| 术语 | 解释 |
|------|------|
| 资源 | 项目中的团队成员，包括平面、后期、物料等类型 |
| 任务 | 需要完成的工作项，有工时、类型、优先级等属性 |
| 依赖关系 | 任务之间的先后顺序关系，前置任务完成后才能开始后续任务 |
| 排期 | 根据任务属性和资源情况，自动计算任务的开始和结束时间 |
| 甘特图 | 一种项目管理图表，横轴为时间，显示任务的安排情况 |
| 复合任务 | 同时包含平面和后期工作的任务 |
| 关键路径 | 影响项目总工期的一系列任务链 |
| 资源冲突 | 同一个人员在同一时间被分配了多个任务 |
| 物料任务 | 物料准备相关的任务，通常作为其他任务的前置条件 |
| App Token | 飞书多维表的访问令牌 |
| Table ID | 飞书多维表中每个数据表的唯一标识 |
