# 飞书多维表访问功能

## 功能说明

本项目支持通过飞书 OAuth 登录后，直接访问您的飞书多维表，类似于 Coze 工作流中的飞书集成功能。

## 使用方式

### 1. 飞书扫码登录

访问 `/feishu-oauth` 页面，使用飞书 App 扫码登录。

登录成功后，系统会：
- 获取您的用户访问令牌 (`user_access_token`)
- 保存登录状态到浏览器
- 获取您的用户信息（姓名、邮箱等）

### 2. 查看多维表列表

登录成功后，您可以：
- 在登录页面点击"我的多维表"按钮
- 或直接访问 `/feishu-bitable-apps` 页面

系统会显示您有权限访问的所有多维表应用，包括：
- 应用图标和名称
- 应用 Token（用于 API 调用）
- 访问权限状态

### 3. 访问多维表数据

每个多维表都可以：
- 点击"打开"按钮，在飞书中查看完整数据
- 复制应用 Token，用于后续的 API 调用

## API 接口

### 获取多维表列表

**端点**: `GET /api/feishu/user/bitable-apps`

**请求头**:
```
Authorization: Bearer {user_access_token}
Content-Type: application/json
```

**响应示例**:
```json
{
  "code": 0,
  "msg": "success",
  "data": {
    "items": [
      {
        "app_token": "bascnxxxxx",
        "name": "我的项目排期表",
        "url": "https://xxxx.feishu.cn/base/bascnxxxxx",
        "icon": {
          "token": "icon_token"
        }
      }
    ],
    "total": 10,
    "has_more": false
  }
}
```

## 与 Coze 工作流的区别

| 特性 | Coze 工作流 | 本项目 |
|------|------------|--------|
| **集成方式** | 预配置的飞书插件 | 使用飞书官方 API |
| **授权流程** | 可视化配置 OAuth | 自定义实现 OAuth |
| **数据访问** | 通过组件操作 | 直接调用 API |
| **灵活性** | 受限于组件 | 完全自定义 |
| **使用场景** | 快速集成、无代码 | 复杂业务、定制化 |

## 技术实现

### OAuth 授权流程

1. **用户扫码**: 使用飞书二维码 SDK 生成登录二维码
2. **获取授权码**: 用户确认后，通过 SDK 的 `message` 事件获取授权码
3. **交换令牌**: 后端使用授权码换取用户访问令牌
4. **保存状态**: 将令牌保存到浏览器 `localStorage`
5. **访问数据**: 使用用户令牌调用飞书多维表 API

### 安全性

- ✅ 使用飞书官方 OAuth 2.0 协议
- ✅ 令牌仅保存在用户浏览器，不存储到服务器
- ✅ 使用 HTTPS 确保传输安全
- ✅ 支持用户主动退出登录，清除本地令牌

## 注意事项

### 权限要求

要访问您的多维表，需要确保：
1. 您是多维表的创建者或有访问权限
2. 飞书应用已获得必要的权限（读取多维表数据）
3. 飞书开放平台的重定向 URL 已正确配置

### 重定向 URL 配置

在飞书开放平台中配置以下重定向 URL：
```
https://kdck9rbnvr.coze.site/feishu-oauth-callback
```

### 令牌有效期

- 用户访问令牌有效期约为 2 小时
- 令牌过期后需要重新登录
- 建议在每次调用 API 前检查令牌是否有效
