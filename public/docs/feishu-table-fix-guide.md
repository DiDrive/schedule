# 飞书表结构快速修复指南

## 人员表字段类型问题

如果您在同步人员数据时遇到 `UserFieldConvFail (code: 1254066)` 错误，这说明飞书多维表中的"姓名"字段类型不正确。

### 错误原因

飞书的"姓名"字段被创建为**成员类型**，但系统尝试写入的是**文本类型**数据。

### 解决方案

请按以下步骤修改飞书人员表的字段结构：

#### 方案1：删除并重新创建"姓名"字段（推荐）

1. 打开飞书多维表，进入**人员表**
2. 点击"姓名"字段名，选择"删除字段"
3. 点击表头右侧的 "+" 按钮，添加新字段
4. 选择字段类型为 **"文本"**
5. 将字段命名为"姓名"
6. 保存

#### 方案2：添加新字段用于显示姓名

如果您不想删除现有字段，可以：

1. 点击表头右侧的 "+" 按钮
2. 添加一个新的 **"文本"** 类型字段
3. 将字段命名为"姓名-文本"或其他名称
4. 联系开发者，告知新的字段名称

## 正确的人员表字段结构

| 字段名 | 字段类型 | 说明 |
|--------|---------|------|
| 人员ID | 文本 | 人员的唯一标识 |
| 姓名 | 文本 | 人员姓名 |
| 类型 | 单选 | 人员类型：人员/物料/设备 |
| 工作类型 | 单选 | 工作类型：平面/后期/物料 |
| 等级 | 单选 | 资源等级：助理/初级/高级 |
| 效率 | 数字 | 效率系数，0.5-2.0 |
| 可用性 | 数字 | 可用性系数，0-1 |

## 常见字段类型对照

### 文本类型

- 姓名
- 任务名称
- 项目名称
- 描述

### 成员类型

- 负责人（排期表）

- 效率
- 可用性

### 日期类型

- 截止日期

### 单选类型
- 状态
- 优先级
- 任务类型
- 资源类型

## 验证字段类型

在飞书多维表中，您可以查看字段类型：

1. 点击字段名称
2. 查看菜单中的字段类型提示
3. 如果类型不正确，选择"修改字段类型"

## 注意事项

- ⚠️ 不要将"姓名"字段设置为"成员类型"，应该是"文本类型"
- ⚠️ 不要将"负责人"字段设置为"文本类型"，应该是"成员类型"
- ⚠️ 日期字段需要使用"日期"或"日期时间"类型
- ✅ 字段名称必须与文档中完全一致（包括中英文）
- ✅ 字段类型必须与表格中的一致
